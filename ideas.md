# IDEAS

- Sería interesante medir que repositorios modifican las plantillas existentes de spec kit para sus propósitos y ver si hay patrones comunes en cómo lo hacen. Esto podría ayudar a mejorar las plantillas oficiales o crear nuevas plantillas más flexibles.

- También se podría analizar si hay repositorios que crean sus propias plantillas desde cero pero que siguen usando el nombre "specify" o algo similar, lo que podría indicar que están inspirados en el proyecto pero no lo están usando directamente.

- Sería útil comparar el número de repositorios que usan Spec Kit (".specify") con el número de repositorios que usan otras herramientas similares, para entender la adopción relativa de Spec Kit en la comunidad (p.e OpenSpec, TSLL, etc).
